# AI Demo Shell 项目说明

## 这个项目是什么

用来制作 **AI 产品高保真可交互 Demo** 的工具。  
你只需要用文字描述业务流程，不需要写任何代码，就能在浏览器中看到完整的演示界面。

---

## 第一次使用前（只需做一次）

### 1. 安装 AI 助手插件

把项目根目录里的 `myy-create-aui.skill` 文件拖进 Cowork，安装后 Claude 会自动知道怎么帮你生成场景。

### 2. 构建演示文件

打开终端（Mac 自带的「终端」App），输入以下两条命令：

```
cd /Users/你的用户名/Desktop/ai-demo-shell-test
npm run build
```

> 第一条命令是进入项目文件夹，把 `你的用户名` 替换成你自己的 Mac 用户名（在终端窗口标题栏可以看到）。  
> 构建完成后显示 `✓ built in XXXms` 表示成功。构建只需做一次，之后每次改了内容再重新执行一次即可。

---

## 预览演示效果

构建完成后，`dist/` 文件夹里的 `index.html` **不能直接双击打开**（浏览器安全限制会导致空白页）。  
有以下两种方式预览：

### 方式一：终端命令（推荐）

在终端里运行：

```
npm run preview
```

终端会显示一个地址（如 `http://localhost:4173`），在浏览器里打开这个地址即可看到演示。  
关闭终端窗口后预览停止，下次需要再运行一次。

### 方式二：VS Code + Live Server（不想用终端）

1. 安装 [VS Code](https://code.visualstudio.com/)
2. 在 VS Code 扩展商店搜索并安装 **Live Server** 插件
3. 用 VS Code 打开项目文件夹，在左侧文件树中找到 `dist/index.html`
4. 右键 → **Open with Live Server**，浏览器自动打开演示

## 日常使用：新建一个演示场景

**在 Cowork 里，用自然语言告诉 Claude 你想做什么，例如：**

> 「帮我做一个 AI 辅助发票审核的演示，采购员上传发票，AI 检查金额和合规性，给出审核结论。」

Claude 会自动生成剧本文件，保存到 `src/scenarios/` 对应目录，刷新浏览器即可预览。

---

## 修改已有场景

找到 `src/scenarios/{场景名}/business-design.md`，直接修改文件里的文字内容：

- **AI 台词**：改 `### AI 台词` 下面的内容
- **按钮文字**：改 `### 操作按钮` 下面的内容  
- **右侧面板展示的数据**：改 `### 右侧面板` 下面的内容
- **常见追问的回答**：改 `# 常见追问` 里对应问题下面的内容

> 注意：`##` 和 `###` 开头的**标题文字本身不能改**，只能改标题下面的内容。

---

## 场景目录

新项目默认不内置业务演示数据。请在 `src/scenarios/` 下为每个业务场景新建独立文件夹，并放入对应的 `business-design.md`、`scenario.manifest.json` 和 `data/demo-data.json`。
