  • 验证运营项目审批通过后同步运营台账。

四、流程路径明细

TC-001｜开发项目正常流程路径
  1. 张三在集团-武汉分公司-信息部下发起产品立项申请与合同审批流程，选择项目类型为“开发项目”。
  2. 系统校验发起人组织、岗位权限和项目基础资料完整性。
  3. 进入项目负责人确认节点，确认开发项目立项信息。
  4. 进入预算校验节点，校验开发项目预算金额与附件。
  5. 审批通过后生成项目立项记录，流程正常结束。
TC-002｜开发项目异常流程路径
  1. 张三发起产品立项申请与合同审批流程，项目类型选择“开发项目”。
  2. 系统进入表单完整性校验，发现缺少必要附件。
  3. 流程在提交前阻断，不进入项目负责人确认节点。
  4. 页面返回缺少附件的提示信息，用户需补齐附件后重新提交。
TC-003｜运营项目正常流程路径
  1. 张三在集团-武汉分公司-信息部下发起产品立项申请与合同审批流程，选择项目类型为“运营项目”。
  2. 系统校验运营项目费用类型、组织权限和项目基础资料。
  3. 进入运营负责人确认节点，确认运营项目审批信息。
  4. 审批通过后同步运营台账，并写入运营项目审批结果。
TC-004｜运营项目边界条件路径
  1. 张三发起产品立项申请与合同审批流程，项目类型选择“运营项目”。
  2. 系统识别当前发起组织与项目归属组织存在差异。
  3. 进入跨组织权限校验，校验用户是否具备目标组织发起权限。
  4. 权限匹配通过后继续流转；若权限不匹配则阻断并提示组织权限不足。

五、调试结论

覆盖完整性已覆盖项目类型为开发项目、运营项目的关键主路径与异常路径。
用户权限张三在集团-武汉分公司-信息部下具备发起产品立项申请与合同审批流程的权限。
建议后续可补充更细颗粒度的金额、组织、附件组合条件，以扩大回归测试覆盖范围。
