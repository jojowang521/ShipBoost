# 使用者必读 · 快速上手

> 产品经理专用。读完这一页，你就知道这个项目怎么用了。

---

## 这个项目是做什么的

你写一份「剧本文件」，描述 AI 专员怎么说话、用户确认什么、右侧展示什么——浏览器就能直接跑出一个可交互的 AI Demo，不需要写任何代码。

---

## 三步上手

**第一步：启动项目**

确保电脑已安装 Node.js（v18 以上），在项目根目录执行一次：

```
npm install
npm run dev
```

打开浏览器访问终端提示的地址（通常是 `http://localhost:5173`）。

**第二步：写你的剧本**

在 `src/scenarios/` 目录下新建一个文件夹（英文命名），里面创建一个 `business-design.md` 文件。

按照本文件夹里的 **`02 填写业务演示剧本.md`** 填写，参考 **`03 查看示例.md`** 看完整效果。

**第三步：刷新看效果**

保存文件，浏览器自动热刷新，首页会出现你的新场景入口。

## 你能改什么 / 不能改什么

### ✅ 你可以自由修改的

全都在 `src/scenarios/{你的场景}/business-design.md` 这一个文件里：

- AI 每个步骤说的台词
- 流程步骤的数量和名称
- 生成物卡片对应的右侧面板标题和说明
- 底部操作按钮的文字
- 右侧展示区的内容（支持文字、列表、表格）
- 常见追问的问题和回答

### ❌ 绝对不能改的

- 剧本里 `## 步骤 N：` 和 `### AI 台词` / `### 右侧面板` 这类标题的**文字本身**（这是系统识别锚点，改了场景会失效）
- `src/` 下的任何代码文件（`.tsx`、`.ts`、`.css`）
- `.cursor/` 文件夹下的任何内容
- `package.json`、`vite.config.ts` 等工程配置

## 用 AI 帮你写剧本

你不需要手动填模板，直接让 AI 帮你生成。

### 用 Cursor

项目已内置 `business-design-writer` Skill，在 Cursor 里打开这个项目，对 AI 说：

```
帮我做一个「XXX」场景，用户是 XXX，解决 XXX 问题。
```

AI 会在 2 轮对话内生成完整的剧本文件，自动写入正确的路径。

### 用 Claude Code / Claude Cowork

项目根目录已有 `CLAUDE.md`，Claude Code 会自动读取并了解项目规则。直接告诉 Claude：

```
帮我在 src/scenarios/ 下新建一个场景，场景名是「XXX」，
AI 专员叫「XXX」，流程是：用户输入 XXX → AI 分析 → 输出 XXX。
```

Claude 会按照 `03-guide-02-script-template.md` 的格式生成文件。

### 用 Codex / Trae / 其他 AI 工具

把以下两个文件的内容粘贴给 AI，再描述你的业务场景：

1. `03-guide-02-script-template.md`（告诉 AI 格式规范）
2. `03-guide-03-example.md`（告诉 AI 怎么填）

然后说：「按照以上格式，帮我写一个 XXX 场景的剧本文件。」

## 遇到问题

| 问题 | 解决方法 |
|------|---------|
| 浏览器空白 | 检查终端有没有报错，重新 `npm run dev` |
| 新场景不出现 | 确认文件路径：`src/scenarios/{英文名}/business-design.md` |
| 台词没更新 | 保存文件后等 1–2 秒，浏览器自动刷新 |
| 右侧面板打不开 | 检查该步骤的 `### 右侧面板` 下是否有实质内容（不能是空或占位文字） |
| 格式出错 | 确认 `## 步骤 N：` 和 `### AI 台词` 等标题没有被修改 |
