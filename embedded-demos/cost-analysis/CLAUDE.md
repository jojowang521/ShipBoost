# AI Demo Shell · CLAUDE.md

> 这个文件供 Claude Code、Claude Cowork 等 AI 工具自动读取，建立项目上下文。

---

## 项目是什么

这是一个供**产品经理（PM）**快速制作 AI 交互 Demo 的工具项目。PM 编写一份 Markdown 格式的「剧本文件」，无需写任何代码，浏览器即可呈现完整的可交互演示。

**当前用户是产品经理，不是开发者。** 所有操作都围绕剧本文件展开。

---

## PM 可以自由修改的文件

只有这一类文件，PM 可以自由创建和编辑：

```
src/scenarios/{场景英文id}/business-design.md
```

例如：
- `src/scenarios/payment-application/business-design.md`
- `src/scenarios/vendor-payment-audit/business-design.md`

每个场景文件夹下只有这一个文件需要 PM 维护。

---

## PM 绝对不能修改的内容

- 剧本文件里所有 `##` 和 `###` 开头的标题文字（系统解析锚点，改了场景会失效）
- `src/` 下的任何 `.tsx`、`.ts`、`.css` 文件（代码）
- `.cursor/` 下的任何文件
- `package.json`、`vite.config.ts` 等工程配置文件

---

## 新建演示场景的步骤

当 PM 要求你新建一个演示场景时，按以下顺序处理：

1. 阅读 `03-guide-02-script-template.md`，了解格式规范（标题锚点、各字段含义）
2. 阅读 `03-guide-03-example.md`，了解完整的填写示范（付款申请智能审核场景）
3. 在 `src/scenarios/` 下新建文件夹，命名规则：英文 kebab-case，如 `payment-application`
4. 在文件夹内创建 `business-design.md`，严格按照模板格式填写

## 剧本文件格式规则（生成内容时必须遵守）

**文件头（frontmatter）** 必须包含四个字段：

```
---
场景名称: （中文，如「付款申请智能审核」）
AI专员名称: （如「成本请款专员」）
AI专员简介: （一句话）
头像色: blue
---
```

**步骤结构**（标题文字不可更改，只能改内容）：

```
## 步骤 N：{名称}         ← N 从 1 开始，名称自定义
### 触发方式              ← 不可改标题
### AI 台词              ← 不可改标题
### 确认节点（可选）      ← 不可改标题，默认写「无」
### 操作按钮              ← 不可改标题
### 右侧面板              ← 不可改标题
```

**确认节点格式**（预留区，默认写「无」）：

```
### 确认节点（可选）

无
```

**操作按钮格式**：

```
### 操作按钮

- 「按钮文字」→ 进入步骤 N
```

**常见追问**（放在所有步骤之后）：

```
# 常见追问

## 用户可能问的问题

AI 的回答内容。

## 与任务无关的问题

AI 的拒绝引导语（必须有这一条）。
```

## AI 台词 Markdown 格式规范

**排版规范（核心）**

| 属性 | 规则 |
|------|------|
| 字号 | 14px（系统统一，无需指定） |
| 标题（加粗行） | `font-weight: 600`，用 `**加粗行**` 单独成段替代 |
| 正文 | `font-weight: 400` |
| 段落间距 | 16px（空一行即可） |
| 列表项间距 | 8px（系统统一） |
| 表格上下间距 | 16px（系统统一） |

**结构约束**

- 禁止使用 `#` `##` `###` 任何 heading 语法——系统已将其降级为正文字号，语义混乱
- 用 `**加粗行**` 单独成段作为标题
- 层级靠空行和列表缩进区分，不依赖标题层级

**允许在台词中使用的元素**

- 普通文本、段落、`**加粗关键字段**`
- 有序列表、无序列表
- Markdown 表格（建议不超过 5 列）
- `---` 分割线

**不建议在台词中使用的元素**

- Emoji（✅ ❌ ⚠️ 🔴 🟡 🔵 📊 等）——与 B2B 调性不符
- 超宽表格、长报告、复杂表单预览——应放到右侧工作台
- HTML 标签

**正确示例**

```markdown
成本请款专员：付款申请资料识别完成，共发现 1 项金额差异和 1 项账户风险。

**识别摘要**

- 高风险 2 个：第 12 条、第 28 条
- 中风险 7 个：第 19、23、31、37、42、55、68 条
- 低风险 3 个：第 8、34、71 条

建议优先关注**第 12 条**（付款节点模糊）和**第 28 条**（隐藏自动续约条款）。
```

## 文案质量要求

- 符合企业 B2B 产品调性：专业、简洁、无口语化
- AI 专员在台词中必须以具名身份出现（如「成本请款专员：」），禁止匿名「AI：」
- 中文正文使用全角标点（，。；：""），数字和英文使用半角
- 每条台词控制在 80–150 字内，超过时拆分多条串行播放
- 常见追问至少 3 条，最后一条必须是「与任务无关的问题」

---

## 禁止事项

- 禁止在剧本文件中写任何代码（TypeScript、JavaScript、CSS 等）
- 禁止修改或删除 `## / ###` 标题文字
- 禁止生成需要后端 API 的内容（Demo 必须纯前端可运行）
- 禁止在剧本中的右侧面板字段写 HTML 或 CSS，用自然语言 Markdown 描述即可
- 禁止在一个步骤里放超过 1 个确认节点
